📦 Top 5 Key Entities

Control Entry Point
Data Sink (Final Output)
LiveKit Cloud
Real-time communication backbone. Routes SIP calls via Vobiz trunk. Dispatches jobs to Inbound/Outbound agents. Central hub for all voice sessions.
Agent Session (Inbound + Outbound)
Two parallel Python workers (run.py). Inbound = Škoda advisor. Outbound = School receptionist. Each loads config from dashboard JSON on every call.
Groq LLM (Llama 3.3-70B)
Ultra-fast inference via OpenAI-compatible API. Powers both agents. Temperature 0.7. Reads system prompt + user prompt dynamically from config.
Dashboard (Next.js)
Control plane for the system. Pages: Dialer, Config, Logs, Leads, Workflows, Integrations, Wallet. Writes to agent_config.json which agents reload per call.
Analytics Engine
On call disconnect, Groq analyzes the full transcript. Outputs: summary, sentiment (Pos/Neg/Neutral), caller_intent. Saves to call_logs.json + leads.csv.

📋 System Summary

The system is a dual-agent AI voice platform where run.py spawns two always-on workers (inbound/outbound) that register with LiveKit Cloud and handle live phone calls via a Vobiz SIP trunk. Each call triggers a fresh config reload from the Next.js Dashboard, meaning prompts, voices, and models can be changed without restarting agents. Post-call, Groq-powered analytics automatically analyze and archive every conversation.
